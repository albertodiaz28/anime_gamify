Plan: MVP Anime Gamify (Angular + NestJS)

Context

Construir un MVP de una página web de anime con sistema de gamificación. Los usuarios podrán registrarse, explorar animes scrapeados desde AnimeFLV, buscarlos/filtrarlos por múltiples criterios, puntuarlos, comentarlos y subir de nivel desbloqueando recompensas en función de los capítulos que vean. El scraping es one-shot (se ejecuta una sola vez para poblar la BD, dado que es un MVP). Stack: Angular (frontend), NestJS (backend), PostgreSQL + TypeORM, JWT auth. Monorepo Nx. El diseño visual se delegará a la skill ui-ux-pro-max.

3. Backend (NestJS) — Módulos

3.1 auth

  • POST /auth/register — email, password (bcrypt), username
  • POST /auth/login — devuelve JWT
  • Guard JwtAuthGuard global con @Public(); Passport + @nestjs/jwt

3.2 users

  • Entity: id, email, passwordHash, username, level, xp, unlockedSkills[], avatarConfig (jsonb), createdAt
  • GET /users/me, PATCH /users/me/avatar

3.3 categories

  • Entity: id, slug, name + seed inicial (acción, romance, shounen, ...)
  • GET /categories

3.4 animes (núcleo)

  • Entity Anime: id, externalId, title, description, coverUrl, totalEpisodes, seasons, status
  • ManyToMany con Category; OneToMany con Episode
  • GET /animes con query params: q, categoryId, minEpisodes, maxEpisodes, language, season, sort, cursor, limit
  • GET /animes/:id (incluye episodios + servidores)
Performance (crítico):
  • Usar TypeORM QueryBuilder desde el inicio, NO repository.find()
  • Filtros como andWhere condicional; language requiere joins a episodes → servers; categoryId join al pivot
  • Paginación por cursor, basada en (sortKey, id)
  • Índices: title (gin trigram), totalEpisodes, server.language, episode.animeId, pivot anime_category
  • Búsqueda fuzzy con extensión pg_trgm
  • DTOs planos (proyección) — no devolver entidades pesadas

3.5 episodes

  • Episode: id, animeId, number, title
  • Server: id, episodeId, name, embedUrl, language (ES, LAT, JP-SUB, ...)

3.6 scraping (CLI one-shot)

  • nx run backend:scrape (nest-commander)
  • Stack obligatorio: puppeteer-extra + puppeteer-extra-plugin-stealth — AnimeFLV está tras Cloudflare; axios+cheerio devuelve 403
  • Configuración: headless: 'new', args --no-sandbox, --disable-blink-features=AutomationControlled, UA rotatorio
  • Esperar networkidle2; tolerar challenge Cloudflare (timeout 30-45s)
  • Reutilizar una única página (cookies persistentes)
  • Cheerio solo sobre page.content() ya renderizado
  • Pipeline: catálogo paginado → detalle → episodios → servidores
  • Idempotente: upsert por externalId
  • Backoff exponencial + delay aleatorio 1.5–3s
  • Fallback: si Cloudflare bloquea persistentemente → Jikan API (MAL no oficial) para metadatos + AnimeFLV solo para servidores

3.7 ratings

  • Entity (userId, animeId, score 1-10) — unique compuesto
  • POST /animes/:id/rating, GET /animes/:id/rating/aggregate

3.8 comments

  • Entity (id, userId, animeId, body, createdAt, parentId)
  • Validación longitud + sanitización XSS

3.9 gamification

  • WatchedEpisode (userId, episodeId, watchedAt) — unique compuesto (evita farm)
  • POST /episodes/:id/watch registra y suma XP
  • LevelService con tabla XP por nivel en shared-constants
  • Entity Skill (id, name, requiredLevel, type, payload jsonb)
  • Evento UserLeveledUp desbloquea skills automáticamente
  • GET /users/me/skills, GET /skills

3.10 Cross-cutting

  • ValidationPipe global, helmet, CORS, @nestjs/throttler
  • Swagger en /api/docs
  • @nestjs/config + .env (nunca commitear)
  • Migrations TypeORM; synchronize: false

4. Frontend (Angular standalone)

apps/frontend/src/app/

├── shared/              # componentes UI reutilizables (ui-ux-pro-max)

│   ├── catalog/
│   ├── anime-detail/
│   ├── profile/
│   └── player/

  • Standalone components (no NgModules)
  • Signals para estado local; servicios con signal() o @ngrx/signals para global ligero
  • HttpInterceptor para JWT; AuthGuard en rutas protegidas

  • Filtros del catálogo sincronizados con query params
  • Tailwind o Angular Material → decisión final de ui-ux-pro-max
